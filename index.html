HTML

  • The head element contains information about the webpage, like headers, title and other various metadata
  • The body element represents the visible content shown to the user
  • The ul element are unordered lists, and li elements are lists
  • We can also use ol elements for ordered lists instead of ul, which will list 1, 2, 3, 4, 5... etc

CSS

  • A margin indicates how much space we want outside of a HTML element
  • Conversely, padding indicates how much space we want INSIDE of a HTML element
  • The . in CSS is called a class selector, e.g. .card selects all of the classes called card and applies the property values to them
  • CSS can be written inline, in the HTML document or in a seperate file (otherwise known as a "stylesheet")
  • Stylesheets must be linked in the header, using the <link> tag. the rel attribute defines the relationship of the link for the HTML document and href is the actual URL/path of the file

Git

  • git remote add origin <repo_url>: used to initialise source code that exists for a new repo
  • git clone <repo_url>: used to clone an existing repo into local workstation
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Remember to make changes into a branch, and then use a pull/merge request to merge the changes into main
  • Also remember to pull from the main branch after pull/merge requests are complete
